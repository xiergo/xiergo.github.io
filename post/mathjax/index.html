hugo显示数学公式

2019-09-13

文章目录

          • 问题描述
          • 成功示例：
          • 解决步骤
          • references
问题描述

用hugo生成的静态网页中的公式显示不出来，无论是 inline （行内公式）的还是 display style（行间公式）。如下面所示：

this is a test inline expression, $x_1+x2=1$ , and the display expression is as following: $$ Y{ig}=\pi{1,i}N{1,ig}+\pi{2,i}N{2,ig}+(1-\pi{1,i}-\pi{2,i})T_{ig} $$ which both fail to show properly.

经过一番google，找到了一些解决办法，我随便挑了一个很多人都提到的方法，通过mathjax。

成功示例：

This is a inline expresion: $Y_{ig}=\pi_{1,i}N_{1,ig}+\pi_{2,i}N_{2,ig}+(1-\pi_{1,i}-\pi_{2,i})T_{ig}$, while this is a display expression:

$$Y_{ig}=\pi_{1,i}N_{1,ig}+\pi_{2,i}N_{2,ig}+(1-\pi_{1,i}-\pi_{2,i})T_{ig}$$
解决步骤

  1. 将 ./themes/theme-name/layouts/ 文件夹里的东西全都拷到 ./layouts/ 下面。

  2. 在 ./layouts/partials/ 创建一个html文件。命名为 mathjax_support.html

  3. 将下面一段代码写入mathjax_support.html

    <script type="text/javascript" async
 src="https://cdn.mathjax.org/mathjax/latest/MathJax.js?config=TeX-AMS-MML_HTMLorMML">
 MathJax.Hub.Config({
 tex2jax: {
   inlineMath: [['$','$'], ['\\(','\\)']],
   displayMath: [['$$','$$']],
   processEscapes: true,
   processEnvironments: true,
   skipTags: ['script', 'noscript', 'style', 'textarea', 'pre'],
   TeX: { equationNumbers: { autoNumber: "AMS" },
        extensions: ["AMSmath.js", "AMSsymbols.js"] }
 }
 });
 MathJax.Hub.Queue(function() {
   // Fix <code> tags after MathJax finishes running. This is a
   // hack to overcome a shortcoming of Markdown. Discussion at
   // https://github.com/mojombo/jekyll/issues/199
   var all = MathJax.Hub.getAllJax(), i;
   for(i = 0; i < all.length; i += 1) {
       all[i].SourceElement().parentNode.className += ' has-jax';
   }
 });
   
 MathJax.Hub.Config({
 // Autonumbering by mathjax
 TeX: { equationNumbers: { autoNumber: "AMS" } }
 });
</script>

  4. 在./layouts/partials/header.html里添加一行代码 {{ partial "mathjax_support.html" . }} ，将上面这行代码添加在 </head> 前面。如下图所示：

  5. 最后，所有的行间公式都用<div></div>包起来，所有的行间公式都用 backticks （就是键盘左上角tab上面，1的左边，esc的下面的那个键）包起来。

  6. ok. 完成上面的就可以成功的显示所有的公式了。方法有很多，不一定非要我这个，而且每个人的版本，电脑系统，hugo下的theme都不一样，具体细节可能会不同，但大致差不了多少。对于我的系统来说，这个是成功的。
